📋 Cómo funciona

Cada reto pide escribir una expresión que dé un valor. Al pulsar Comprobar, se evalúa la última expresión y se compara con lo esperado: ✅ si está bien, ❌ con una explicación si no. No hace falta return.

Reto 1

El padre de un elemento

Obtener el id del elemento padre de #item-b.

Pista (lección 02) El padre de un elemento se obtiene con .parentElement, y su id con .id.

Reto 2

Cantidad de hijos

Obtener cuántos hijos (elementos) tiene #lista.

Pista (lección 02) Los hijos elemento están en .children; la cantidad, en .children.length. También existen .firstElementChild y .lastElementChild.

Reto 3

El hermano siguiente

Obtener el texto del hermano siguiente de #item-a.

Pista (lección 02) El hermano siguiente es .nextElementSibling (el anterior, .previousElementSibling); su texto se lee con .textContent.

Reto 4

El primer hijo

Obtener el texto del primer hijo de #lista (firstElementChild).

Pista (lección 02) El primer hijo elemento es .firstElementChild; el último, .lastElementChild.

Reto 5

El hermano anterior

Obtener el texto del hermano anterior de #item-c (previousElementSibling).

Pista (lección 02) El hermano anterior es .previousElementSibling (puede ser null si es el primero).

Reto 6 · combinar
