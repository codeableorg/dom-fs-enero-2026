Leer y escribir el contenido

Una vez que tienes un elemento, lo más común es leer o cambiar su contenido. Hay tres propiedades para esto y se confunden mucho. Esta es la diferencia:

PropiedadQué devuelve / escribe¿Interpreta HTML?
textContent Solo el texto, sin etiquetas No (lo escribe tal cual, como texto)
innerHTML El HTML interno (etiquetas incluidas) Sí (crea elementos reales)
outerHTML El elemento completo, incluyéndose a sí mismo Sí (puede reemplazar el elemento entero)

Partiendo de este HTML:

<p id="saludo">Hola <strong>mundo</strong></p>
AccesoResultado
saludo.textContent"Hola mundo"
saludo.innerHTML"Hola <strong>mundo</strong>"
saludo.outerHTML"<p id=\"saludo\">Hola <strong>mundo</strong></p>"
Cuidado con innerHTML Si metes texto que escribió un usuario dentro de innerHTML, ese texto se interpreta como HTML. Eso abre la puerta a inyecciones (XSS). Cuando solo necesitas mostrar texto, usa textContent: es más seguro y más rápido.

La jerarquía: de dónde sale cada propiedad

En el navegador, todo objeto del DOM hereda de otros más generales. Saber esta cadena te explica por qué un elemento tiene tantos métodos: los va heredando. 

EventTarget        ← lo más general: "puedo recibir eventos"
   ↑
  Node             ← "soy un nodo del árbol" (texto, comentario, elemento…)
   ↑
  Element          ← "soy una etiqueta" (atributos, querySelector…)
   ↑
  HTMLElement      ← "soy una etiqueta HTML" (style, dataset…)
   ↑
  HTMLInputElement ← clase específica de <input> (value, checked…)

Qué aporta cada nivel

NivelQué representaEjemplos que aporta
EventTarget Cualquier cosa que escucha eventos addEventListener, removeEventListener, dispatchEvent
Node Un nodo cualquiera del árbol textContent, childNodes, parentNode, appendChild
Element Un nodo que es una etiqueta innerHTML, outerHTML, getAttribute, classList, querySelector
HTMLElement Una etiqueta del estándar HTML style, dataset, hidden, click()
Clases específicas Cada etiqueta con sus rasgos propios HTMLInputElement.value, HTMLAnchorElement.href, HTMLImageElement.src
Lectura útil Fíjate que textContent vive en Node, pero innerHTML y outerHTML viven en Element. Por eso un nodo de texto tiene textContent pero no tiene innerHTML: no es un elemento.

No todo nodo es un elemento

El DOM tiene varios tipos de nodo. Los tres que más vas a encontrar:

nodeTypeTipoEjemplo
1Elemento<div>, <p>
3Nodo de textoel "Hola" dentro de <p>Hola</p>

Esto explica una sorpresa típica: childNodes (de Node) incluye espacios y saltos de línea como nodos de texto, mientras que children (de Element) solo trae los elementos. Por eso casi siempre se prefiere children.
