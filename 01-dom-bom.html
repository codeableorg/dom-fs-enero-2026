🌳 ¿Qué es el DOM?

DOM significa Document Object Model (Modelo de Objetos del Documento). Cuando el navegador carga tu HTML, lo convierte en un árbol de objetos que JavaScript puede leer y modificar.

Cada etiqueta HTML se convierte en un nodo de ese árbol:

document
 └── html
      ├── head
      │    └── title
      └── body
           ├── h1
           └── p

El punto de entrada al árbol es el objeto global document. Desde ahí puedes alcanzar cualquier elemento de la página.

📌 Idea clave El DOM no es tu archivo HTML. Es la representación viva que el navegador construye en memoria. Si JavaScript cambia el DOM, la página cambia al instante (aunque el archivo .html no se modifique).

    🪟 ¿Qué es el BOM?

    BOM significa Browser Object Model (Modelo de Objetos del Navegador). Mientras el DOM representa la página, el BOM representa el navegador que rodea a esa página: la ventana, la URL, el historial, la pantalla…

    El objeto principal del BOM es window, y dentro tiene:

    ObjetoPara qué sirveEjemplo
    windowLa ventana del navegador (objeto global)window.innerWidth
    navigatorInformación del navegador y del dispositivonavigator.language
    locationLa URL actual (y permite navegar)location.href
    historyEl historial de páginas visitadashistory.back()
    screenLa pantalla física del usuarioscreen.width
    ⚠️ Diferencia DOM vs BOM DOM = el contenido de la página (etiquetas, texto, estilos).
    BOM = el navegador alrededor (ventana, URL, historial, pantalla).
    Curiosidad: document en realidad vive dentro de window, así que el DOM es como "una parte" de lo que controla el BOM.
