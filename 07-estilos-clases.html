7 · Estilos y clases

Dar estilo a un elemento desde JavaScript: la propiedad style, el camelCase de CSS, classList y getComputedStyle.

Volver al índice

Dos caminos para dar estilo

Desde JS puedes pintar un elemento de dos formas. La mayoría de las veces conviene la segunda:

CaminoCómoCuándo
style (en línea) Propiedad por propiedad: el.style.color = "red" Ajustes puntuales y dinámicos (una posición, un alto calculado…)
classList (clases) Enciendes/apagas clases definidas en el CSS Lo habitual: el estilo vive en el CSS, JS solo cambia clases

La propiedad style (estilos en línea)

elemento.style escribe estilos en línea (los del atributo style="..."). Dos reglas de oro: cada propiedad va en camelCase y cada valor es un string con sus unidades ("16px", no 16). 

caja.style.color = "white";
caja.style.backgroundColor = "#1e40af";  // camelCase, y el valor es texto
caja.style.padding = "20px";             // recuerda las unidades

Elemento de prueba

Cámbiame los estilos en línea.
Atajos útiles caja.style.cssText = "color:white; background:navy" aplica varias de golpe con sintaxis CSS normal. Y caja.style.setProperty("background-color", "navy") usa el nombre con guion (imprescindible para variables como --azul).

El camelCase de las propiedades CSS

En CSS las propiedades llevan guion (background-color). En JS los guiones no valen como nombre de propiedad, así que se escriben en camelCase: se quita el guion y la siguiente letra va en mayúscula.

CSSJS (style.)
background-colorbackgroundColor
font-sizefontSize
border-radiusborderRadius
margin-topmarginTop
z-indexzIndex
floatcssFloat (caso especial)

Pulsa cada botón: aplica una propiedad y te muestra su traducción CSS → JS.

Elemento de prueba

Observa cómo cambio con cada propiedad.
¿No te sale el camelCase? También puedes usar corchetes con el nombre exacto de CSS: caja.style["background-color"] = "navy". Funciona igual, pero el camelCase es lo habitual.

className vs classList

className es un string con todas las clases. Si le asignas un valor, reemplazas todo lo que hubiera, y es fácil borrar sin querer una clase importante. classList es una API para tocar una clase sin pisar las demás.

Elemento de prueba (empieza con dos clases)

caja-estilo + destacado
El peligro de className Al hacer caja.className = "caja-estilo" se pierde destacado (y cualquier otra clase). Para sumar o quitar una sola, usa classList.

classList: add, remove, toggle, replace, contains

Cinco métodos que cubren casi todo. Cada botón ejecuta uno y verás el efecto al instante:

MétodoQué hace
add("x")Agrega la clase (si ya está, no pasa nada)
remove("x")Quita la clase
toggle("x")Si está la quita, si no está la pone. Devuelve true/false
replace("a","b")Cambia a por b (solo si a estaba)
contains("x")Pregunta si la tiene: true o false

getComputedStyle: leer el estilo REAL

element.style solo ve lo que está en línea. Si el color viene de tu archivo CSS, element.style.color sale vacío. Para conocer el valor realmente aplicado (venga de donde venga) se usa getComputedStyle(elemento).

Resumen rápido

  • style aplica estilos en línea: propiedades en camelCase y valores como string con unidades.
  • CSS con guion → JS en camelCase (background-color → backgroundColor).
  • className reemplaza TODAS las clases; classList toca una sin pisar el resto.
  • classList: add, remove, toggle, replace y contains.
  • getComputedStyle lee el estilo real aplicado (solo lectura, valores resueltos).
