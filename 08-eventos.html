8 · Eventos

Cómo los elementos y la ventana avisan de que «algo pasó» y cómo reaccionar con funciones (handlers).

Volver al índice

¿Qué es un evento?

Un evento es un aviso de que ocurrió algo: un clic, una tecla, mover el ratón, enviar un formulario, que la página terminó de cargar… Tanto los nodos del DOM como window y document pueden emitir eventos, porque todos heredan de EventTarget (lo viste en la jerarquía EventTarget → Node → Element de la lección 4).

Para reaccionar, le damos al elemento una función llamada handler (o listener): «cuando ocurra este evento, ejecuta esta función».

El patrón siempre es el mismo Eliges en qué elemento escuchar, qué tipo de evento ("click", "keydown"…) y qué función ejecutar cuando pase.

Tres formas de registrar un handler

FormaCómoPega
1. En el HTML <button onclick="hola()"> Mezcla HTML y JS; solo admite un handler
2. Propiedad .onclick boton.onclick = hola Solo UNA función: asignar otra la pisa
3. addEventListener boton.addEventListener("click", hola) La recomendada: admite VARIOS handlers y opciones

Formas 1 y 2: atributo HTML y propiedad onclick

La forma 1 es la que han usado los botones de todo el curso: onclick dentro de la etiqueta. La forma 2 asigna una función a la propiedad .onclick desde JS; ojo, solo guarda una: si asignas otra, reemplaza a la anterior.

Forma 3: addEventListener (la recomendada)

elemento.addEventListener(tipo, funcion, opciones). Su gran ventaja: puedes registrar varios handlers para el mismo evento y ninguno pisa al otro.

boton.addEventListener("click", registrar);  // los DOS se ejecutan

La opción once

Con { once: true } el handler se ejecuta una sola vez y se quita solo. (Otras opciones útiles: capture, que verás en la lección 9, y passive.)

Listener de un solo uso

Quitar un handler: removeEventListener

Para quitar un listener necesitas pasar exactamente la misma función que usaste al agregarlo. Por eso debe ser una función con nombre (guardada en una variable): una función anónima no se puede quitar, porque no tienes su referencia. 

function alPulsar() { /* ... */ }
boton.addEventListener("click", alPulsar);
boton.removeEventListener("click", alPulsar);  // misma referencia: se quita

Activa y quita el listener del botón objetivo

El objeto Event

Cuando ocurre un evento, el navegador llama a tu handler y le pasa un objeto event lleno de información: qué tipo fue, dónde ocurrió, coordenadas del ratón, qué tecla se pulsó, etc. 

boton.addEventListener("click", function (event) {
  console.log(event.type);    // "click"
  console.log(event.target);  // el elemento donde ocurrió
});

preventDefault(): cancelar la acción por defecto

Algunos eventos tienen una acción por defecto del navegador: un enlace navega, un formulario se envía (y recarga la página). event.preventDefault() cancela esa acción para que la controles tú con JS.

Si desmarcas la casilla, la página se recargará Es justo lo que hace un formulario por defecto: enviarse y recargar. Por eso casi siempre se usa preventDefault() para manejar el envío con JS sin perder la página.

Resumen rápido

  • Todo lo que herede de EventTarget (nodos, window, document) emite eventos.
  • Tres formas de escuchar: atributo HTML, propiedad .onclick y addEventListener (la mejor).
  • addEventListener admite varios handlers y opciones como { once: true }.
  • removeEventListener exige la misma función con nombre que registraste.
  • El handler recibe un objeto event con type, target, coordenadas, tecla…
  • preventDefault() cancela la acción por defecto (navegar, enviar un formulario…).
