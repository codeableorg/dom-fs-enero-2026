5 · Atributos y propiedades

Cómo el HTML que escribes (atributos) se ve desde JavaScript (propiedades).

Atributo no es lo mismo que propiedad

Cuando escribes <input id="correo" value="hola">, eso son atributos en el HTML. Al cargar la página, el navegador crea un objeto y le pone propiedades equivalentes. Casi siempre coinciden, pero no son lo mismo.

Atributo (HTML) Propiedad (JS)
Dónde vive En el texto del HTML En el objeto del DOM
Cómo se accede getAttribute() / setAttribute() elemento.propiedad
Valor Siempre texto (string) Puede ser texto, número o booleano
Al escribir el usuario No cambia (queda el valor inicial) Sí cambia (refleja lo actual)
Regla práctica Para leer el estado actual de un campo, usa la propiedad (input.value). El atributo (getAttribute("value")) suele quedarse con el valor inicial que venía escrito en el HTML.

Nombres que cambian: className y htmlFor

Algunas palabras ya están reservadas en JavaScript (class, for), así que la propiedad equivalente lleva otro nombre. Estas dos son las que más confunden:

Atributo HTML Propiedad en JS Por qué
class className class es palabra reservada de JS
for (en <label>) htmlFor for es palabra reservada (los bucles)

En general, los atributos con guion pasan a camelCase en JS. Por ejemplo maxlength se vuelve maxLength, y tabindex se vuelve tabIndex.

className vs classList className es la cadena completa de clases (la reemplazas entera). Para agregar o quitar una sola clase es más cómodo classList.add(), classList.remove() y classList.toggle().

Atributos booleanos

Atributos como disabled, checked, readonly o required son booleanos: lo que importa es si están o no están, no su valor. En HTML basta con escribir el nombre: 

<button disabled>No puedes pulsarme</button>
<input type="checkbox" checked>

Escribir disabled="false" no lo activa: con que el atributo aparezca, ya cuenta como verdadero. Desde JS, en cambio, la propiedad es un booleano real: 

boton.disabled = true;   // lo desactiva
boton.disabled = false;  // lo activa
caja.checked;            // true o false según esté marcada

Acepto los términos

Atributos no estándar y data-*

Si inventas un atributo que no existe en HTML (por ejemplo color-favorito), el navegador no crea una propiedad para él. Tendrás que leerlo con getAttribute(): 

// <div id="x" color-favorito="azul">
x.colorFavorito         // undefined (no hay propiedad)
x.getAttribute("color-favorito")   // "azul"

La forma correcta de guardar tus propios datos en el HTML es con atributos data-*. Esos sí tienen un acceso cómodo desde JS a través de dataset: 

<article data-id="42" data-precio-final="1200">

articulo.dataset.id            // "42"
articulo.dataset.precioFinal   // "1200"  (data-precio-final → precioFinal)
De guion a camelCase data-precio-final se lee como dataset.precioFinal. Todo lo que sale de dataset es texto; si necesitas un número, conviértelo con Number(...).

La propiedad style

element.style da acceso a los estilos en línea del elemento (los del atributo style="..."). Cada propiedad CSS se escribe en camelCase: background-color se vuelve style.backgroundColor. 

caja.style.color = "white";
caja.style.backgroundColor = "navy";  // ojo: camelCase
caja.style.padding = "12px";
style solo ve lo «en línea» element.style.color devuelve vacío si el color viene de tu archivo CSS y no de un style="...". Para saber el color realmente aplicado se usa getComputedStyle(elemento). Además, para muchos casos es más limpio cambiar de clase con classList que tocar style propiedad por propiedad.

Resumen rápido

  • Atributo (HTML, texto) y propiedad (JS, valor real) suelen coincidir, pero no son lo mismo.
  • getAttribute/setAttribute para atributos; punto (el.prop) para propiedades.
  • class → className, for → htmlFor, guiones → camelCase.
  • Booleanos: en HTML basta que aparezcan; en JS son true/false.
  • Datos propios: usa data-* y léelos con dataset.
  • style solo ve lo en línea; lo aplicado real se ve con getComputedStyle.
